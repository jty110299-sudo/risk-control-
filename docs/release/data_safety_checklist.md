# Data Safety Checklist

* [ ] No large raw official datasets committed.
* [ ] No real personal identifiers.
* [ ] No real internal business data.
* [ ] No generated fake risk result presented as real.
* [ ] Public data sources documented.
* [ ] Derived data attribution preserved.
* [ ] Synthetic supplements labeled.
* [ ] `data/raw` ignored.
* [ ] `data/processed` ignored unless approved.
* [ ] `outputs` reviewed before commit.
