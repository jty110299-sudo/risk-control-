# Agent Positioning

## Purpose

Define the product position for Risk Control Agent.

## Product Name

Risk Control Agent

## Target Users

* Risk analysts
* Model monitoring analysts
* Risk operation teams
* Data analysts

## User Pain Points

* Risk metrics are often scattered across dashboards and reports.
* Analysts need to distinguish business change, data issues, and model degradation.
* Segment contribution analysis can be repetitive and error-prone.
* Reports need consistent structure, evidence, and safety wording.
* High-risk operational suggestions require clear human-review boundaries.

## Agent Value Proposition

Risk Control Agent is planned to help users organize monitoring evidence, detect abnormal changes, structure analysis, and draft auditable risk reports.

Risk Control Agent should not only provide backend analysis outputs. It should eventually provide an analyst-facing dashboard so that risk analysts can inspect alerts, metrics, root causes, model stability, data quality, and generated reports.

## Core Capabilities Planned for Later Phases

* Public-dataset-based monitoring data preparation.
* Metric calculation and comparison.
* Alert rule detection.
* Segment decomposition.
* Model monitoring summary.
* Data quality and drift checks.
* Risk report drafting.
* Analyst-facing Dashboard / UI for reviewing metrics, alerts, root causes, and reports.
* Output and log saving under `outputs/`.

## Current Phase Limitation

The current project remains in documentation, data source design, and product design phases. It does not implement UI code, backend business code, data generation, model training, or an executable Agent workflow.
