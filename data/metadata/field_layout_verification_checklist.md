# Field Layout Verification Checklist

No data processing should begin until file layout verification is complete.

## Checklist

| Item | Status | Notes |
| --- | --- | --- |
| Confirm official user guide version | Not started |  |
| Confirm file layout version | Not started |  |
| Confirm delimiter | Not started |  |
| Confirm encoding | Not started |  |
| Confirm header availability | Not started |  |
| Confirm loan identifier field | Not started |  |
| Confirm reporting period field | Not started |  |
| Confirm origination/static fields | Not started |  |
| Confirm monthly performance fields | Not started |  |
| Confirm delinquency status field | Not started |  |
| Confirm zero balance / default / loss-related fields if available | Not started |  |
| Confirm missing value encoding | Not started |  |
| Confirm data type assumptions | Not started |  |
| Confirm whether sample file matches official layout | Not started |  |
| Confirm mapping to Agent monitoring tables | Not started |  |

## Verification Principle

The selected file layout must match the selected source file version before any ingestion, cleaning, metric calculation, alerting, report generation, or UI implementation begins.
